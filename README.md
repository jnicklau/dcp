# Stochastic Battery Optimisation with Bayesian Neural Networks

Receding-horizon MPC battery scheduler that integrates probabilistic load and price forecasts with a stochastic LP. Uncertainty is modelled via MC Dropout or Bayes by Backprop; scenarios are drawn with temporally correlated noise (Cholesky decomposition of the residual correlation matrix).

---

## Overview

```
BNN forecasts (DLA consumption, spot price, waste heat)
        │
        ▼
Scenario sampling  ──  Cholesky-correlated Gaussian draws
        │
        ▼
Extensive-form stochastic LP  (CVXPY / ECOS)
        │
        ▼
MPC receding-horizon execution  →  cost comparison vs. baseline
```

Three signals are forecast independently:
| Signal | Unit | Model |
|---|---|---|
| DLA power consumption | kWh / 15 min | `GaussianMCDNet` / `GaussianBNNNet` |
| Spot price (Germany EXAA) | EUR / MWh | `GaussianMCDNet` / `GaussianBNNNet` |
| Nestlé waste heat | MW | `GaussianMCDNet` / `GaussianBNNNet` |

## Usage

### 1 — Train

```bash
python train_simple.py
```

Trains BNN models for all three signals, estimates the Cholesky residual correlation matrices, prints test-set metrics, and saves:
- `models/dla_bnn.pt`, `models/price_bnn.pt`, `models/abwaerme_bnn.pt`
- `norms/dla_norm.npz`, `norms/price_norm.npz`, `norms/abwaerme_norm.npz`
- `norms/dla_chol.npz`, `norms/price_chol.npz`

Key options in `config.py`:

| Parameter | Default | Description |
|---|---|---|
| `UNCERTAINTY_METHOD` | `"mcd"` | `"mcd"` = MC Dropout, `"bnn"` = Bayes by Backprop |
| `NUM_EPOCHS` | `10` | Training epochs |
| `HIDDEN_DIMS` | `[64, 64, 32]` | Hidden layer widths |

### 2 — Evaluate and visualise

```bash
python eval_visualize.py
```

Runs the full evaluation pipeline for week 48 (configurable inside the file) and saves:
- `week{N}_predictions.png` — forecast vs. actual with aleatoric/epistemic uncertainty bands
- `week{N}_optimization.png` — SOC trajectory, schedule, and cost comparison

### 3 — Custom optimisation run

```python
from run_optimization import run_optimization
results = run_optimization(week_num=10)
```

## Configuration

All tunable parameters live in `config.py`. Key switches:

| Parameter | Options | Effect |
|---|---|---|
| `UNCERTAINTY_METHOD` | `"mcd"`, `"bnn"` | Epistemic uncertainty method |
| `SCENARIO_METHOD` | `"cholesky_decomp_corr"` | Scenario sampling method |
| `SO_MODE` | `"extensive"`, `"scenario_avg"`, `"mean_scenario"` | Optimisation mode (see below) |
| `HORIZON_HOURS` | `12` | MPC look-ahead window in hours |
| `N_SCENARIOS` | `10` | Scenarios per LP solve |
| `OPT_FREQUENCY` | `12` | Steps committed per MPC iteration (3 h = 12 * 4 (four 15min intervals per hour)) |


# Lag features

All three signals use lags at 48, 96, and 672 steps (12 h, 24 h, 1 week) to capture diurnal and weekly seasonality. Configured via `DLA_LAG_STEPS`, `PRICE_LAG_STEPS`, `ABWAERME_LAG_STEPS` in `config.py`.

### Optimisation modes

| Mode | Description |
|---|---|
| `"extensive"` | Correct stochastic programming: all scenarios share one schedule; per-scenario SOC constraints enforced |
| `"mean_scenario"` | Single deterministic LP on expected inputs — fastest |
| `"scenario_avg"` | Independent LP per scenario, schedules averaged — legacy |

## Using the optimizer with external scenarios

The optimizer is designed so that scenario generation and scheduling are fully decoupled. You do not need BNN forecasts to use the LP — any source of (price, consumption) trajectories works.

### Public API

```python
from stoch_optimizer import StochasticOptimizer

opt = StochasticOptimizer(battery_capacity=500, max_power=100, efficiency=0.9)

# --- Option A: bring your own scenarios ---
# Each scenario is a tuple (prices, consumption), both 1-D arrays of length horizon_steps.
my_scenarios = [
    (price_array_1, consumption_array_1),
    (price_array_2, consumption_array_2),
    ...
]
result = opt.optimize(my_scenarios, current_soc=250.0, discharge_allowed=True, mode="extensive")

# --- Option B: sample from BNN mean/std (built-in Cholesky sampler) ---
scenarios = opt.sample_scenarios(price_mean, price_std, dla_mean, dla_std)
result = opt.optimize(scenarios, current_soc=250.0)

# --- Option C: one-liner convenience wrapper ---
result = opt.optimize_from_stats(price_mean, price_std, dla_mean, dla_std, current_soc=250.0)
```

`result` always has the same structure regardless of which path was taken:

```python
{
    "expected_cost":  float,         # mean cost across scenarios (EUR)
    "cost_std":       float,         # std of costs across scenarios
    "avg_soc":        np.ndarray,    # SOC trajectory, shape (horizon_steps+1,)
    "avg_charge":     np.ndarray,    # charge schedule, shape (horizon_steps,)
    "avg_discharge":  np.ndarray,    # discharge schedule, shape (horizon_steps,)
}
```
